# Prompt Engineering Comparison: Context vs No Context

## Objective

Evaluate the differences between a roadmap generated using:

1. Prompt only (No Context)
2. Prompt + User Context

---

# Output 1: Prompt Without Context

## Prompt

```text
Create a 30-day learning roadmap.

Include:
- Weekly milestones
- Daily tasks
- Resources
- Projects
- Final outcome

Make it practical and beginner-friendly.
```

## Generated Roadmap Summary

### Weekly Milestones

| Week | Focus |
|------|--------|
| Week 1 | Foundations — syntax, variables, loops, functions |
| Week 2 | Data structures — lists, dictionaries, files, libraries |
| Week 3 | Real projects — APIs, scripts, automation |
| Week 4 | Portfolio — polish, deploy, present work |

### Projects

| Week | Project |
|------|----------|
| Week 1 | Number Quiz Game |
| Week 2 | To-Do List App |
| Week 3 | Weather Dashboard |
| Week 4 | Personal Portfolio Script |

### Final Outcome

- Write clean Python scripts from scratch
- Build 3 portfolio-ready projects
- Become comfortable reading and debugging code
- Be ready for intermediate Python topics

### Analysis

#### Strengths

- Beginner-friendly structure
- Covers fundamental programming concepts
- Clear progression from basics to projects
- Easy for a wide audience to follow

#### Limitations

- Generic roadmap
- No personalization
- No career alignment
- Limited industry-specific relevance

# Output 2: Prompt With Context

## Additional Context

```text
User Role: Data Scientist

Generate a roadmap aligned with data science skills,
portfolio development, and practical industry workflows.
```

## Generated Roadmap Summary

### Weekly Milestones

| Week | Focus |
|------|--------|
| Week 1 | Data Wrangling with Pandas |
| Week 2 | Data Visualization & EDA |
| Week 3 | Machine Learning Fundamentals |
| Week 4 | End-to-End Portfolio Project |

### Projects

| Week | Project |
|------|----------|
| Week 1 | Titanic Dataset Deep Dive |
| Week 2 | World Happiness Report EDA |
| Week 3 | House Price Predictor |
| Week 4 | End-to-End Portfolio Notebook |

### Final Outcome

- Clean and transform real datasets
- Build insightful visualizations
- Train and evaluate machine learning models
- Create a professional portfolio project
- Develop industry-relevant data science skills

### Analysis

#### Strengths

- Personalized roadmap
- Career-oriented project selection
- Uses real-world datasets
- Portfolio-focused learning path
- Mimics practical industry workflows

#### Limitations

- Assumes Python fundamentals
- More specialized audience
- Requires domain-specific tools and libraries

# Comparative Analysis

| Aspect | Prompt Only | Prompt + Context |
|----------|------------|------------------|
| Personalization | Low | High |
| Career Alignment | Generic | Data Science Specific |
| Portfolio Value | Moderate | High |
| Industry Relevance | Limited | Strong |
| Learning Efficiency | Moderate | High |
| Real-World Application | Basic | Extensive |
| Dataset Usage | None | Real Datasets |
| Professional Readiness | Moderate | Strong |

---

# Key Learnings

- Context significantly improves output relevance.
- Generic prompts produce broad educational content.
- User-specific context enables targeted roadmap generation.
- Personalized projects increase practical value.
- Context-aware prompts create stronger portfolio opportunities.
- Career-aligned roadmaps improve learning efficiency.
- Real-world datasets make outputs more applicable.
- Additional context reduces ambiguity and improves specificity.

---

# Observations

## Prompt Without Context

- Generated a general Python learning roadmap.
- Focused on foundational programming skills.
- Suitable for users without a defined career goal.
- Prioritized coding concepts over domain application.

## Prompt With Context

- Generated a data science-focused roadmap.
- Included industry-standard tools and workflows.
- Emphasized portfolio development.
- Leveraged real-world datasets and machine learning concepts.

## Overall Observation

Providing user context transformed the roadmap from a generic programming guide into a career-oriented learning plan. The generated content became more relevant, actionable, and aligned with professional objectives. The inclusion of domain-specific projects and industry workflows substantially increased the roadmap's practical value.
