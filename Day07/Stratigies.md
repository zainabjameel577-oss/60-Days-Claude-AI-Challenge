# Claude Usage Strategy
*Personalized for Coding, Research, Learning, and Student Workflows*

---

# Recommended Primary Model

## Claude Sonnet (Daily Driver)

**Best balance of speed, reasoning, and quality.**

Use Sonnet for:
- Coding projects
- Debugging
- Assignment help
- Research summaries
- Paper analysis
- Learning new concepts
- Project architecture
- General productivity

### Why Sonnet?
Sonnet provides:
- Strong reasoning ability
- Good explanation quality
- Fast response times
- Efficient usage limits
- Reliable coding assistance

For most users, Sonnet handles **80–90% of tasks effectively**.

---

# Effort Levels Strategy

Claude's effort settings matter almost as much as model selection.

| Effort Level | When to Use | Frequency |
|-------------|------------|------------|
| Low | Simple lookups, syntax questions | Rarely |
| Standard | Most daily tasks | Default |
| High | Complex debugging, deep analysis, architecture | Frequently when stuck |
| Max | Critical academic or professional work | Occasionally |

### Rule of Thumb

Start with:

> Sonnet + Standard

Upgrade only when:
- The problem is difficult
- The answer lacks depth
- Multiple iterations are failing
- Research synthesis is required

# Biggest Mistakes to Avoid

## 1. Using Opus for Everything

Problem:
- Consumes limits unnecessarily
- Often provides minimal benefit on routine tasks

Better:
- Use Sonnet for most work
- Upgrade only when necessary

---

## 2. Asking Vague Questions

Bad:
> Fix my code.

Good:
> Here's my code, expected output, actual output, and error message.

More context produces significantly better results.

---

## 3. Skipping Effort Upgrades

Many difficult tasks fail because effort remains at Standard.

Upgrade to High when:
- Debugging difficult issues
- Analyzing research
- Designing systems
- Performing root-cause analysis

---

## 4. Accepting the First Answer Blindly

For important work:
- Ask for reasoning
- Request alternatives
- Challenge assumptions
- Ask for trade-off analysis

Iterative conversations often outperform one-shot prompts.

# Final Recommendation

## Default Configuration

**Model:** Sonnet  
**Effort:** Standard

This combination covers the majority of:
- Coding
- Learning
- Research
- Student work
- General productivity

## Upgrade Path

| Situation | Model | Effort |
|-----------|--------|---------|
| Quick lookup | Haiku | Low |
| Daily work | Sonnet | Standard |
| Hard debugging | Sonnet | High |
| Research synthesis | Sonnet | High |
| Algorithm design | Opus | High |
| Thesis/dissertation | Opus | Max |

### Simple Decision Rule

1. Start with **Sonnet + Standard**
2. If stuck, switch to **Sonnet + High**
3. If the work is academically or professionally critical, use **Opus**
4. Reserve **Max effort** for your most important projects

This maximizes quality while preserving usage limits and response speed.
